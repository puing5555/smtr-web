## Part 1: QA 체크리스트 (QA Agent 지시용)

### 🔴 A. 시그널 데이터 품질 (매번 필수 검증)

#### A1. 비종목 필터링
- **발견된 문제:** 비트코인 67건, 이더리움 9건, 금 3건, 천연가스 1건 등 비종목이 시그널로 추출됨
- **체크:** ticker가 암호화폐(BTC, ETH, XRP 등), 원자재(금, 은, 구리, 천연가스), 통화(달러, 엔) 포함 여부
- **조치:** 전량 제거

#### A2. 비상장기업
- **발견된 문제:** 스페이스X, 오픈AI, 웨이모, 야놀자, 삼성파운드리, 삼성디스플레이, 화웨이 등 비상장 ticker
- **체크:** ticker가 실제 상장 종목인지 (Yahoo Finance에서 조회 가능한지)
- **조치:** 상장 안 된 것은 제거 또는 관련 상장 ticker로 대체 (예: 삼성파운드리→삼성전자)

#### A3. key_quote 품질
- **발견된 문제 1:** 자동자막 오류 텍스트 그대로 들어감 — "제승당 순위 투하즈 도리스 탑 백호 asos 모스 와이어드"
- **발견된 문제 2:** key_quote가 20자 미만 (너무 짧아 의미 전달 안 됨)
- **발견된 문제 3:** key_quote가 200자 초과 (너무 길어 핵심 안 보임)
- **체크항목:**
  - 한글+영어 랜덤 혼합 패턴 감지 (자동자막 오류)
  - 문맥 없는 단어 나열 감지
  - 길이 범위: 20자~200자
  - 실제 한국어 문장인지 (조사, 어미 포함)
- **조치:** 심각한 건 삭제, 경미한 건 summary 기반 재작성

#### A4. signal_date (날짜)
- **발견된 문제:** 과거 영상인데 signal_date가 오늘 날짜(2026-03-03)로 들어감
- **체크:** signal_date가 영상 업로드 날짜와 일치하는지
- **체크:** 미래 날짜 없는지
- **체크:** 모든 시그널이 같은 날짜면 의심 (배치 INSERT 버그)
- **조치:** 영상 업로드 날짜로 수정

#### A5. timestamp (타임스탬프)
- **발견된 문제:** DB subtitle_text가 5000자로 잘려서 key_quote 매칭 실패 → timestamp "00:00"
- **체크:** timestamp가 "00:00"인 시그널 비율 (10% 이상이면 의심)
- **체크:** 같은 영상의 모든 시그널이 같은 timestamp면 의심
- **조치:** subs/ 폴더 원본 자막에서 key_quote 기반 재매칭

#### A6. 중복 체크
- **발견된 문제:** 같은 영상 + 같은 종목 중복 3건
- **체크:** (video_id + stock_name) 조합 중복
- **체크:** (video_id + ticker) 조합 중복
- **체크:** key_quote 내용이 거의 동일한 시그널
- **조치:** 중복 제거 (최신 or 높은 confidence 유지)

#### A7. signal_type 유효성
- **프로젝트 규칙:** 매수/긍정/중립/경계/매도 (한글)
- **체크:** 영문 BUY/SELL 없는지
- **체크:** 빈값 없는지
- **조치:** 한글 규칙으로 통일

#### A8. confidence 범위
- **체크:** 0~1 사이 값
- **체크:** 모든 시그널이 같은 confidence면 의심
- **조치:** 범위 밖이면 수정

### 🟡 B. 프론트엔드/사이트 검증

#### B1. 종목 페이지 생성
- **발견된 문제:** 새 종목 시그널 INSERT 후 빌드했는데 종목 페이지 안 생김 (NVO)
- **체크:** DB에 있는 모든 ticker에 대해 /stock/{ticker} 페이지 존재하는지
- **체크:** 빌드 로그에서 생성된 stock 페이지 수 확인
- **조치:** 누락 종목 페이지 재빌드

#### B2. 인플루언서 페이지
- **발견된 문제:** 새 채널 추가 후 인플루언서 페이지 404 (월가아재)
- **원인:** SPEAKER_SLUGS 매핑에 없어서 generateStaticParams에서 누락
- **체크:** DB speakers 테이블의 모든 speaker에 대해 /profile/influencer/{slug} 존재하는지
- **조치:** SPEAKER_SLUGS 매핑 추가 + 재빌드

#### B3. 모달 UI 일관성
- **발견된 문제:** 종목 상세 모달과 탐색 페이지 모달 UI가 다름 (마크다운 렌더링 vs 텍스트)
- **체크:** 모달에서 "## 투자포인트" 같은 마크다운이 텍스트로 노출되지 않는지
- **체크:** AiDetailRenderer 컴포넌트가 모든 모달에 적용됐는지
- **조치:** 컴포넌트 통일

#### B4. 가격 데이터
- **발견된 문제:** SQ 가격 누락, NVO 스키마 불일치 (price vs current_price)
- **체크:** stockPrices.json에 모든 종목 포함
- **체크:** 가격 스키마 일관성 (current_price/last_updated 통일)
- **체크:** 가격이 0이거나 비정상적인 값
- **조치:** Yahoo Finance에서 재수집 + 스키마 통일

#### B5. HTTP 상태
- **체크:** 메인, explore, my-stocks 페이지 200 OK
- **체크:** 모든 종목/인플루언서 페이지 200 OK (404 없는지)
- **조치:** 404 페이지 재빌드

### 🟢 C. 애널리스트 리포트 검증

#### C1. 애널리스트 이름
- **발견된 문제:** "되었음을", "자의", "-" 등 의미없는 텍스트가 애널리스트 이름으로 들어감 (311건 오류)
- **원인:** PDF 파서가 애널리스트명을 잘못 추출
- **체크:** analyst_name이 한글 2~4글자 이름인지
- **체크:** "-", 빈값, 1글자, 5글자 이상, 의미없는 단어
- **조치:** PDF 원본에서 재추출 또는 증권사 기반 추정

#### C2. AI 요약 포맷
- **발견된 문제:** ai_detail이 **볼드** 형식, 【괄호】형식, **섹션:** 형식 등 불일치
- **정규화:** 모든 ai_detail은 ## 헤더 형식 (투자포인트/실적전망/밸류에이션/리스크/결론)
- **체크:** ## 헤더 5개 섹션 존재하는지
- **체크:** 마크다운 렌더링 정상인지
- **조치:** 텍스트 치환으로 정규화

## Part 2: 프롬프트 고도화 지침 (Prompt Agent 지시용)

### 현재 프롬프트: pipeline_v10.md (V10.8)

### 🔴 반드시 추가해야 할 규칙

#### P1. 비종목 필터링 규칙 추가
```
현재 문제: 비트코인, 이더리움, 금, 천연가스 등이 종목으로 추출됨
추가할 규칙:
- 다음은 종목이 아님: 비트코인(BTC), 이더리움(ETH), 리플(XRP), 솔라나(SOL) 등 모든 암호화폐
- 다음은 종목이 아님: 금(GLD), 은(SLV), 원유(WTI), 천연가스, 구리 등 원자재
- 다음은 종목이 아님: 달러, 엔, 유로 등 통화
- ticker가 없는 비상장기업(스페이스X, 오픈AI 등)은 시그널 제외
```

#### P2. key_quote 품질 기준 강화
```
현재 문제: 자동자막 오류 텍스트가 key_quote로 들어감
추가할 규칙:
- key_quote는 반드시 의미 있는 한국어 문장이어야 함
- 자동자막 오류 패턴 감지: 영어+한글 랜덤 혼합, 조사 없는 단어 나열
- 자동자막 품질이 나쁜 영상은 summary에서 핵심 문장 추출하여 key_quote 대체
- key_quote 길이: 최소 20자, 최대 200자
- key_quote에 종목명이 반드시 포함되어야 함
```

#### P3. signal_date 규칙
```
현재 문제: 오늘 날짜가 signal_date로 들어감
추가할 규칙:
- signal_date는 반드시 영상 업로드 날짜 사용
- 절대 현재 날짜(today) 사용 금지
- 영상 메타데이터의 upload_date 또는 publishedAt 사용
```

#### P4. 중복 방지
```
현재 문제: 같은 영상에서 같은 종목 시그널 중복 추출
추가할 규칙:
- 하나의 영상에서 같은 종목은 1개 시그널만 추출
- 여러 번 언급되면 가장 핵심적인 1개만 선택
- confidence가 높은 것 우선
```

### 🟡 개선하면 좋을 규칙

#### P5. ticker 정확도
```
현재 문제: 폭스바겐→ticker 없음, 루멘텀→ticker 없음, BYD→ticker 없음
추가할 규칙:
- 외국 종목은 반드시 정확한 ticker 포함 (폭스바겐→VOW3.DE, BYD→1211.HK)
- ticker를 모르면 시그널 제외하지 말고 stock_name만 기록하고 ticker는 빈칸
- 한국 종목은 6자리 코드 (삼성전자→005930)
```

#### P6. 매크로/교육 영상 필터링
```
현재 문제: 월가아재 채널 436개 중 개별 종목 시그널은 15개뿐 (나머지 매크로/교육)
추가할 규칙:
- 영상에서 개별 종목을 직접 분석하지 않으면 시그널 없음으로 처리
- "시황", "매크로", "교육", "기초강의" 등 키워드 포함 시 종목 시그널 추출 주의
- 종목이 단순 예시로 언급된 것 vs 실제 투자 의견이 있는 것 구분
```

#### P7. confidence 기준
```
추가할 규칙:
- confidence는 발언의 확신도를 기반으로
- 0.9+: 명확한 매수/매도 추천 ("이 종목 지금 사야 합니다")
- 0.7-0.9: 긍정적/부정적 의견 ("좋아 보입니다", "주의가 필요합니다")
- 0.5-0.7: 중립적 언급 ("이 종목은 이런 상황입니다")
- 0.5 미만: 시그널 제외
```

### 🟢 프롬프트 정리/삭제 대상

#### P8. 불필요한 규칙 정리
```
점검 대상:
- V10에 누적된 중복 규칙 제거
- 더 이상 사용하지 않는 필드 정의 제거
- 너무 상세한 예시 → 핵심만 남기고 축소
- 프롬프트 총 길이를 최소화 (토큰 비용 절감)
```

## Part 3: 시스템 설정 관련 QA (Patrol Agent)

### S1. OpenClaw 설정
- openclaw.json에 "ignoreBots" 키 없는지 확인 (있으면 게이트웨이 안 뜸)

- 세션 토큰 사용량 모니터링 (200k 근처면 경고)

### S3. 프로세스 감시
- 게이트웨이 포트 18789 LISTENING 확인
- heartbeat 정상 여부
- Agent failed 에러 감지

## Part 5: QA Agent한테 보낼 첫 지시 (복붙용)

```
QA 상시 검증 작업 시작해:

1. 신규 INSERT 전 필수 검증:
   - 비종목 필터 (암호화폐/원자재/통화 제거)
   - 비상장기업 ticker 제거
   - key_quote 품질 (자동자막 오류 감지, 20~200자)
   - signal_date = 영상 업로드 날짜 (오늘 날짜 금지)
   - 중복 체크 (video_id + ticker)
   - signal_type 한글 규칙 (매수/긍정/중립/경계/매도)

2. 통과 기준:
   - 95%↑ 통과
   - 80-95% 부분수정
   - 80%↓ 전체재작업

3. 검증 결과 포맷:
   - 총 시그널 수
   - 항목별 이슈 건수
   - 통과율 %
   - 오류 목록 (구체적: 어떤 데이터가 왜 틀렸는지)
   - 심각도 분류 (🔴긴급 🟡주의 🟢참고)
```
