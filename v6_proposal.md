# V5→V6 프롬프트 개선 제안서

## 분석 기반
- 삼프로TV 19영상 42시그널 (V4→V5 적용 경험)
- 코린이아빠 25시그널 (V4 재검증 경험)
- 채널 유형별 엣지 케이스 추론 (슈카월드/개인유튜버/코인/ETF/뉴스형)

---

## 발견된 문제 패턴

### 패턴 1: 발언자 이름 변형 (Name Aliasing)
- **발생**: 삼프로TV
- **예시**: 박명성 = 박명석 = 글로벌 시황맨 (자막 자동생성에서 이름이 다르게 표기)
- **V5 판단**: 별도 인물로 처리 → 중복 제거 누락
- **올바른 판단**: 동일 인물 → 중복 제거 대상
- **제안**: 발언자 정규화 규칙 추가. "별명/직함으로도 검색"

### 패턴 2: 지수/섹터 시그널 분류 불명확
- **발생**: 삼프로TV
- **예시**: 배재원 "코스피 7000 갈 수 있다", 고연수 "증권주 스트롱바이", 이재규 "건설주 재평가"
- **V5 판단**: 종목처럼 처리 (1영상×1종목)
- **올바른 판단**: 지수(코스피) ≠ 종목, 섹터(증권주) = 유효하되 별도 표기
- **제안**: market 필드에 INDEX/SECTOR 구분, 지수 전망은 시그널 제외

### 패턴 3: "리포트 + 미묘한 동의" 경계선
- **발생**: 삼프로TV (차영주)
- **예시**: 
  - (a) "교보증권에서 37만원 상향했습니다" → 리포트 전달 ✓
  - (b) "이수페타시스... 장기 투자자에게 솔깃한 뉴스" → 리포트 + 미약한 동의?
  - (c) "에코프로머티... 영업이익률 10%는 상당히 의미 있는 수치" → 긍정 뉘앙스
- **V5 판단**: 전부 리포트→중립으로 처리
- **올바른 판단**: (b)(c)는 미묘하지만 V5 기준이 명확("본인 돈 넣겠다는 뉘앙스")이므로 중립 유지가 맞음
- **제안**: 현행 유지 + confidence=low 표기로 애매한 케이스 추적

### 패턴 4: 교육형/설명형 채널 오분류 위험
- **발생 예상**: 슈카월드, 김단테
- **예시**: 
  - 슈카 "엔비디아가 이렇게 된 이유는..." (설명) → V5가 결론으로 오분류 가능
  - 김단테 "비트코인 100K 시나리오가 있다" (분석) → 전망인지 권유인지 불명확
- **V5 판단**: 결론/매수로 과대 분류 위험
- **제안**: "교육/설명" mention_type 추가 → 시그널 제외

### 패턴 5: 조건부 발언 처리 미흡
- **발생**: 삼프로TV, 코린이아빠
- **예시**: 
  - 김장열 "1분기 실적까지는 업사이드 > 다운사이드" (시간 조건)
  - 박병창 "가격 상승률 둔화되면 주의" (조건부 경고)
  - 코린이아빠 "이 가격대에서는 매수" (가격 조건)
- **V5 판단**: 조건 무시하고 최종 의견만 채택
- **올바른 판단**: 조건 존재 시 confidence=medium으로 하향
- **제안**: 조건부 발언 → confidence 자동 하향 + reasoning에 조건 명시

### 패턴 6: 동일 영상 내 의견 변화
- **발생**: 삼프로TV
- **예시**: 김장열 삼성전자 "목표주가 33만원(긍정)... 하지만 신규 매수는 신중(중립)"
- **V5 판단**: 최종 의견 기준 = 중립
- **문제**: 기존 보유자 vs 신규 진입자에 따라 다른 시그널
- **제안**: 보유자/신규진입 구분이 명확하면 2개 시그널 허용 (reasoning에 명시)

### 패턴 7: 크로스에셋 채널의 비교 분석
- **발생 예상**: 김단테 (코인+주식), 솔튜브 (ETF+개별주)
- **예시**: "비트코인보다 이더리움이 나아 보인다" → 비트코인=경계? 이더리움=긍정?
- **V5 판단**: 각각 별도 시그널
- **문제**: 상대적 비교를 절대적 시그널로 변환 시 왜곡
- **제안**: 비교 발언 → 선호 종목만 시그널 (비선호 종목은 시그널 제외)

### 패턴 8: ETF 추천의 종목 분류
- **발생 예상**: 솔튜브, 박현지(삼프로TV)
- **예시**: "TIGER 반도체 ETF 사라" → 개별 종목 시그널? ETF 시그널?
- **V5 판단**: ETF를 종목처럼 처리
- **제안**: market=ETF 구분, ETF 시그널은 구성종목으로 전파하지 않음

### 패턴 9: 시황맨/뉴스 앵커의 이중 역할
- **발생**: 삼프로TV (박명성)
- **예시**: 
  - "삼성전자 2027년 매출 세계 최고" (뉴스 전달) → 중립
  - "엔비디아 5% 급락은 과도, 없는 분들에게 기회" (본인 의견) → 매수
- **V5 판단**: 같은 사람이지만 영상마다 역할이 다름
- **문제**: 시황 보도 코너에서 간간이 개인 의견 삽입
- **제안**: 시황/뉴스 코너 → 기본값 뉴스→중립, 명시적 권유 표현 있을 때만 상향

### 패턴 10: 자동생성 자막의 종목명 오류
- **발생**: 삼프로TV 전반
- **예시**: "최영주" = 차영주, "LG 이너택" = LG이노텍, "SKSQ" = SK스퀘어
- **V5 판단**: 자막 그대로 사용 → 종목명 매칭 실패 위험
- **제안**: 종목명 정규화 단계 추가 (Step 2.5)

### 패턴 11: 목표주가 제시 = 매수인가?
- **발생**: 삼프로TV (비차영주 전문가들)
- **예시**: 박병창 "삼성전자 목표주가 34만원" → 이것이 매수 권유인가?
- **V5 판단**: 규칙 불명확 (전문가가 본인 의견으로 TP 제시 vs 리포트 TP 전달)
- **제안**: 본인 TP 제시 + 긍정 톤 → 긍정, 본인 TP + "사라" 표현 → 매수, TP만 = 리포트

### 패턴 12: 1인 유튜버의 "썸네일/제목 과장" vs 실제 내용
- **발생 예상**: 개인 유튜버 전반
- **예시**: 제목 "삼성전자 올인합니다!!!" → 실제 내용은 "관심 가져볼 만하다"
- **V5 판단**: 자막 기준 분석이므로 제목은 무관
- **제안**: 명시적으로 "제목/썸네일은 시그널 판단에 사용하지 않음" 규칙 추가

## V6 규칙 개선안

### 신규 규칙 추가

#### V6-R1: 발언자 정규화 (Name Normalization)
```
자막 자동생성에서 이름이 다르게 표기될 수 있음.
- 동일 인물 판단 기준: 소속+직함+코너명이 같으면 동일 인물
- 예: "박명성" = "박명석" = "글로벌 시황맨" (같은 인물)
- 중복 제거 시 정규화된 이름 기준으로 적용
```

#### V6-R2: 지수/섹터/ETF 분류
```
market 필드 확장:
- KR / US / CRYPTO / SECTOR / INDEX / ETF

처리 규칙:
- INDEX (코스피, S&P500 등): 시그널 제외, 시장 전망으로만 기록
- SECTOR (증권주, 건설주 등): 시그널 유지, market=SECTOR 표기
- ETF (TIGER 반도체 등): 시그널 유지, market=ETF 표기, 구성종목으로 전파 금지
```

#### V6-R3: 교육/설명형 콘텐츠 구분
```
mention_type 확장: 결론/논거/뉴스/리포트/교육

"교육" = 현상 설명, 원리 해설, 역사적 맥락 제공 (투자 의견 없음)
→ 시그널 = "해당없음" (테이블에서 제외)

판단 기준:
- "~인 이유는", "~의 구조는", "~을 알아보자" = 교육
- 교육 후 "그래서 나는 ~한다" = 결론 (교육 아님)
```

#### V6-R4: 조건부 발언 처리
```
조건부 발언 감지 시:
- "~한다면", "~될 경우", "이 가격대에서는", "1분기까지는"
→ confidence = medium (무조건)
→ reasoning에 조건 원문 포함

시간 조건이 만료된 경우:
→ 시그널 유효성 재검토 대상으로 표기
```

#### V6-R5: 자막 종목명 정규화 (Step 2.5)
```
자막 추출 직후, 시그널 판단 전에 종목명 정규화 수행:
- 음성 인식 오류 교정 (이너택→이노텍, 하이니스→하이닉스)
- 약칭 → 정식명 (삼전→삼성전자, 하닉→SK하이닉스)
- 영문 → 한글 매칭 (NVIDIA→엔비디아)
```

#### V6-R6: 제목/썸네일 무시 규칙
```
영상 제목과 썸네일은 시그널 판단에 사용하지 않음.
오직 자막(발언 내용)만 기준.
제목이 "올인합니다!!!"여도 실제 발언이 "관심 가져볼 만"이면 → 긍정
```

### 기존 규칙 수정

#### V5-R2 수정: 리포트 브리핑 강화
```
기존: (a) 전달만 → 리포트/중립, (b) 전달+동의 → 결론
추가: (d) 전달 + 미묘한 긍정 뉘앙스 ("의미 있다", "긍정적이다")
→ mention_type = "리포트", 신호 = "중립", confidence = "low"
→ reasoning에 "미묘한 동의 뉘앙스 존재하나 명시적 추천 아님" 기록
```

#### V5-R3 수정: 전망 vs 권유 판단 강화
```
기존 기준 유지 + 추가:
- "충분히 가능하다고 봅니다" = 전망 (긍정) ← 새로 명시
- "올라갈 수밖에 없다" = 전망 (긍정) ← 강한 전망도 전망
- "지금 안 사면 후회한다" = 권유 (매수) ← 간접적이지만 매수 권유
- "못 사면 어쩔 수 없다, 사면 좋겠다" = 전망 (긍정) ← 희망 표현

핵심: 동사 어미로 판단
- ~이다/된다/보인다/전망이다 = 전망
- ~해라/사라/담아라/들어가라/늘려라 = 권유
- ~하시면 좋겠다/추천드립니다 = 권유 (공손한 형태)
```

#### V5-R5 수정: 게스트 채널 규칙 확장
```
기존: 진행자 질문 ≠ 신호, 게스트 답변만 신호
추가:
- 시황/뉴스 앵커: 기본값 = 뉴스/중립, 명시적 권유 시만 상향
- 마감시황 코너 복수 게스트: 각각 분리하되, 패널 토론에서
  "저도 동의합니다"는 독립 시그널이 아닌 기존 게스트 시그널의 confidence 보강
```

### V6 교차 검증 추가 항목
```
기존 6개 + 새로 4개:
7. 발언자 이름 정규화 완료 여부
8. 지수(INDEX) 시그널이 포함되지 않았는지
9. 교육/설명 콘텐츠가 시그널로 잡히지 않았는지
10. 조건부 발언의 confidence가 medium 이하인지
```

## V4→V5→V6 변경 요약

| 규칙 | V4 | V5 | V6 |
|------|-----|-----|-----|
| 등급 | 5단계 (봇 임의 생성) | 5단계 엄격 강제 | 5단계 유지 |
| 리포트 브리핑 | 구분 없음 | 전달 vs 본인 의견 | + 미묘한 동의 → confidence=low |
| 시장 전망 | 구분 없음 | 전망 vs 권유 구분 | + 동사 어미 판단 기준 명시 |
| 중복 처리 | 없음 | 같은 발언자+종목 | + 이름 정규화 후 중복 제거 |
| mention_type | 결론/논거/뉴스 | +리포트 (4종) | +교육 (5종) |
| market | KR/US/CRYPTO | +SECTOR | +INDEX, +ETF |
| confidence | 없음 | high/medium/low | + 조건부→자동 medium |
| 종목명 | 자막 그대로 | 자막 그대로 | Step 2.5 정규화 |
| 제목 사용 | 암묵적 | 암묵적 | 명시적 제외 |
| 교차 검증 | 6항목 | 6항목 | 10항목 |
