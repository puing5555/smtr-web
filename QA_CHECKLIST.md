# QA 배포 전 필수 체크리스트
_Created: 2026-03-04 | Updated: 2026-03-04_

> **이 체크리스트는 배포 전 필수 검증 항목임. 1개라도 실패하면 배포 차단.**
> QA 서브에이전트는 이 파일을 반드시 읽고 전체 항목 검증 후 리포트 제출.

---

## 3단계 QA 프로세스

```
INSERT → QA(데이터) → 빌드 → QA(페이지) → 배포 → QA(사이트)
```

- **Stage 1 (데이터)**: INSERT 후, 빌드 전 실행
- **Stage 2 (페이지)**: 빌드 후, 배포 전 실행
- **Stage 3 (사이트)**: 배포 후 실행

---

## [A. 데이터 정합성] — Stage 1

- [ ] DB 시그널 총 수 = 프론트 표시 수 (limit 하드코딩으로 잘리는 거 없는지)
- [ ] 인플루언서 탭: DB 채널 수 == UI 표시 채널 수
- [ ] 각 채널 카드 클릭 시 시그널 수 일치 확인
- [ ] 날짜 필드 = 영상 업로드 날짜 (created_at/INSERT 날짜 아님)
  - → 샘플 10개의 날짜가 영상 실제 날짜와 일치하는지 확인
- [ ] published_at NULL인 레코드 없는지 확인

## [B. 종목 페이지] — Stage 2

- [ ] DB ticker 고유 수 == 생성된 종목 페이지 수 (out/stock/*.html)
- [ ] 누락 종목 페이지 있으면 **배포 차단**
- [ ] 차트보기 링크 샘플 10개 클릭 → 404 없는지 확인
- [ ] stockPrices.json에 모든 종목 가격 데이터 있는지

## [C. 영상 링크/타임스탬프] — Stage 2

- [ ] 피드 카드 "영상보기" 링크에 &t={seconds} 파라미터 포함 확인
- [ ] 모달 "영상보기" 링크 타임스탬프 정상 여부
- [ ] 샘플 5개 영상링크 직접 클릭 → 시그널 발언 위치 도달 확인
- [ ] timestamp 0초 또는 비정상값 없는지

## [D. UI 렌더링] — Stage 3

- [ ] 메인 피드 로드 → HTTP 200 + 시그널 카드 표시
- [ ] 인플루언서 탭 → **전체 채널** 표시 (1개만 나오면 실패)
- [ ] explore 페이지 정상
- [ ] my-stocks 페이지 정상
- [ ] 종목 상세 모달 → 차트 + 핵심발언 + 영상링크 정상

## [E. INSERT 후 필수 프로세스] — Stage 1~2

- [ ] INSERT → 빌드 → 배포 (이 순서 필수, 빌드 안 하면 새 종목 페이지 안 생김)
- [ ] 빌드 후 종목 페이지 수 = DB ticker 수 재확인
- [ ] stockPrices.json 신규 종목 가격 포함 확인

## [F. 스키마/데이터 품질] — Stage 1

- [ ] 가격 필드명 통일 (price vs current_price 같은 불일치 없는지)
- [ ] 티커별 가격 데이터 존재 여부 (NULL/누락 없는지)
- [ ] 시그널 필수 필드 전부 채워져 있는지 (ticker, signal, key_quote, timestamp)

---

## 통과 기준

- **전체 통과**: 배포 진행
- **1개라도 실패**: 배포 차단 → Dev 수정 → 재검증
- **긴급 hotfix 아닌 이상 QA 미통과 배포 금지**

## QA 리포트 표준 형식

```
[QA 리포트 - YYYY-MM-DD HH:MM]
Stage: 1(데이터) / 2(페이지) / 3(사이트)
체크리스트: N/N 통과 ✅ (또는 N/N 통과 ❌)
실패 항목:
- [B-1] DB ticker 151개, 페이지 54개 → 97개 누락
판정: 배포 진행 ✅ / 배포 차단 ❌ → Dev 수정 필요
```

## 교훈 (2026-03-04)

| 버그 | QA가 놓친 이유 |
|------|--------------|
| 이효석만 표시 | "전체 채널 표시 여부" 체크 없음 |
| 날짜 전부 오늘 | "날짜 = 영상 날짜인지" 검증 안 함 |
| 영상보기 → 처음 이동 | "타임스탬프 도달 확인" 안 함 |
| 차트보기 → 404 | "종목 페이지 존재 여부" 검증 안 함 |
