## 발견된 주요 문제 패턴

### 1. 시그널 유효성 문제
- **초단기 매매**: "지금 들어가!"라는 발언의 시그널 가치
- **시간 민감성**: 방송 시점과 실제 시청 시점의 괴리
- **유효기간**: 라이브에서의 "지금"은 녹화본에서 의미 없음

### 2. 1영상 1시그널 규칙 충돌
- **같은 종목 반복**: 삼성전자에 대해 1영상 내 BUY→SELL→BUY
- **해결 방안 필요**: 최종 의견으로 통합? 시간순 마지막? 가중평균?

### 3. 시청자 vs 방송자 의견 구분
- **채팅 반응**: "채팅에서 LG전자 물어보셨는데 저는 글쎄요..."
- **질문 답변**: 방송자 본인 의견 vs 시청자 질문에 대한 단순 답변
- **신호 가치**: 시청자 질문에 대한 답변을 시그널로 인정할지?

### 4. 자막 정확도 문제
- **실시간 자동자막**: 종목명 오인식 빈발
- **전문용어**: "코스피", "코스닥" → "고스피", "고스닥"  
- **후처리 필요**: 종목명 사전 기반 보정 로직 필요

### 5. 교육 vs 매매신호 구분
- **교육성 설명**: "일반적으로 이런 패턴에서는..."
- **실제 매매신호**: "지금 이 타이밍에 들어가세요"
- **mention_type 분류**: 둘 다 "교육"으로 분류되기 쉬움

## 잘 작동한 V6 규칙

### 1. mention_type 분류
- **결론** vs **논거** 구분 잘 됨
- "삼전 지금 들어가!" (결론) vs "삼전이 지지선을 뚫고 올라와서" (논거)

### 2. market 분류  
- **KR** 시장 종목들 정확히 식별
- **SECTOR** (반도체, 자동차) vs **INDEX** (코스피) 구분 잘 됨

### 3. 타임스탬프 추출
- 정확한 발언 시점 캡처 가능
- 영상 시작부분(00:00) 회피 잘 됨
