## 문제 패턴 식별

### 1. 전략 vs 종목 시그널 혼재
- **문제**: "모멘텀 전략이 유효하다" vs "삼성전자 매수"
- **해결**: 전략 평가는 SECTOR/교육, 종목 추천은 KR/결론으로 구분

### 2. 스크리닝 결과의 시그널 중복
- **문제**: 한 영상에서 5개 종목 언급 시 5개 시그널 생성
- **해결**: 동일 논리/근거면 대표 종목 1개만 시그널로 추출

### 3. 통계적 표현의 신뢰도 판단
- **문제**: "승률 68%"를 confidence로 어떻게 반영할 것인가?
- **해결**: 70% 이상 → high, 60-70% → medium, 60% 미만 → low

### 4. 자산배분 권고의 시그널 분류
- **문제**: "주식 비중 70%로 확대" → 매수 시그널인가?
- **해결**: MACRO 시장에서 자산군별 긍정/부정 신호로 처리

### 5. 교육 vs 결론의 구분 모호성
- **문제**: 백테스트 결과가 교육인지 투자 결론인지 애매
- **해결**: 과거 분석 → 교육, 현재/미래 권고 → 결론

## 잘 작동한 V6 규칙

### 1. Market 분류 명확성
- KR/US/CRYPTO 구분이 명확했음
- SECTOR/MACRO 분류도 적절히 작동

### 2. mention_type 구분 효과성
- 결론 vs 논거 구분이 퀀트 채널에서도 유용
- 교육 카테고리가 백테스트/통계 설명에 적합

### 3. 신호 5단계의 적정성
- 매수/긍정/중립/경계/매도 분류가 퀀트 결론에도 적용 가능
- 통계적 근거가 있는 권고에 잘 매핑됨

## 권고사항

### 1. 퀀트 특화 가이드라인 추가
```
- 백테스트 결과만 제시 → 교육
- 백테스트 + 현재 적용 권고 → 결론
- 팩터/스타일 분석 → SECTOR
- 포트폴리오 비중 조정 → MACRO
- 통계적 유의성 70% 이상 → confidence: high
```

### 2. 스크리닝 결과 처리 규칙
```
- 동일 영상 동일 논리 → 1개 대표 시그널
- 서로 다른 섹터/논리 → 각각 시그널
- 리스트 형태 제시 → mention_type: 결론
```

### 3. Confidence 레벨 매핑
```
- 백테스트 승률 70%+ → high
- 통계적 검증 60-70% → medium  
- 일반적 분석/의견 → low
- 정량적 근거 없음 → low
```

## 요약

퀀트/데이터 분석형 채널에서 V6 파이프라인은 **70% 수준으로 적절히 작동**했으나, 다음 영역에서 개선이 필요:

1. **전략 vs 종목 구분**: 투자전략 평가와 개별 종목 추천의 명확한 분리 필요
2. **교육 vs 결론 판단**: 과거 분석(교육)과 현재 권고(결론)의 구분 기준 정립
3. **스크리닝 결과 처리**: 다수 종목 동시 언급 시 시그널 중복 방지 로직 필요
4. **통계적 신뢰도**: 정량적 근거의 confidence 레벨 자동 판단 기준 수립

특히 **SECTOR와 MACRO 분류가 퀀트 채널에 매우 적합**했으며, **mention_type 중 '논거'가 데이터 기반 근거 제시에 효과적**이었음.
