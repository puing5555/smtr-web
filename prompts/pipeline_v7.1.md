# 인플루언서 파이프라인 프롬프트 V7.1 (최종)

---

## 텔레그램 봇용 통합 프롬프트

```
아래 유튜브 채널의 영상을 분석해줘.
채널: {CHANNEL_URL}

════════════════════════════════════
STEP 1: 최근 영상 30개 목록 (제목, URL, 길이)
════════════════════════════════════

STEP 1.5: 제목 사전 필터링
- pass: 특정 종목 언급, 시장/섹터 전망, 매매 의견, 경제지표의 시장 영향
- skip: 구독자 Q&A, 일상/브이로그, 채널 공지, 투자 무관 교양/역사
⚠️ 제목은 필터링에만 사용. 시그널 판단에는 절대 사용 금지 (제목 과장 문제)

════════════════════════════════════
STEP 2: 통과한 영상 자막 추출 (한국어)
════════════════════════════════════

════════════════════════════════════
STEP 2.5: 자막 종목명 정규화
════════════════════════════════════
자막 추출 직후, 분석 전에 종목명 정규화 수행:
- 음성인식 오류 교정: 이너택→LG이노텍, 하이니스→SK하이닉스, 최영주→차영주
- 약칭→정식명: 삼전→삼성전자, 하닉→SK하이닉스, 현차→현대차
- 영문→한글 매칭: NVIDIA→엔비디아, TSLA→테슬라
- 인물명 정규화: 같은 소속+직함+코너명이면 동일 인물
  예: "박명성" = "박명석" = "글로벌 시황맨" → "박명성"으로 통일

════════════════════════════════════
STEP 3: 전체 영상 상세 요약 (각 영상마다)
════════════════════════════════════
- 최소 500자 이상 상세 요약
- 영상 흐름 (시작 → 논거 전개 → 결론)
- 반론/반박 구조 명시
- 조건부 발언 명시 (조건 원문 포함)
- 확신 정도 (강하게 추천 vs 조심스럽게)
- ⭐ 영상의 최종 결론 명확히 기술
- ⭐ 각 종목의 언급 목적 분류 (8종):
  - "결론" = 발언자 본인이 투자 의견을 제시하는 종목
  - "논거" = 다른 종목의 투자 논리를 뒷받침하기 위한 비교/예시
  - "뉴스" = 단순 시황/뉴스 전달 (본인 투자 의견 없음)
  - "리포트" = 증권사 리포트/목표주가 전달
  - "교육" = 현상 설명, 원리 해설, 역사적 맥락 (투자 의견 없음)
  - "티저" = 유료 콘텐츠/멤버십 전환을 위한 미끼 (V7 신규)
  - "보유" = 과거/현재 보유 경험 언급, 현재 매매 의견 아님 (V7 신규)
  - "컨센서스" = 복수 증권사 목표주가/의견 종합 전달 (V7 신규)

STEP 3.5: 발언자 추출
- 채널명 아닌 실제 발언자 이름
- 진행자와 게스트 구분
- 게스트 여러 명이면 각각 분리
- ⭐ 발언자 이름 정규화 적용 (STEP 2.5에서 정의)

════════════════════════════════════
STEP 4: 신호 판단
════════════════════════════════════

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⭐ 신호 5단계 (이것만 사용. 다른 등급 사용 금지)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔵 매수: "사라, 담아라, 들어가라, 비중 확대" (명확한 매수 액션 권유)
🟢 긍정: "좋아보인다, 괜찮다, 관심가져라" (호의적이나 명확한 매수 권유 아님)
🟡 중립: "지켜보자, 모르겠다" 또는 뉴스/리포트/교육 전달
🟠 경계: "조심해라, 리스크 있다, 주의" (부정적이나 매도 권유 아님)
🔴 매도: "팔아라, 손절해라, 빠져라" (명확한 매도 액션 권유)

❌ 절대 사용 금지: STRONG_BUY, STRONG_SELL, HOLD, BUY, SELL
   등 위 한글 5개(매수/긍정/중립/경계/매도)만 사용!
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

═══════════════════════
핵심 규칙 1: 논거 vs 결론 구분
═══════════════════════
- "결론 종목": 발언자가 실제 투자 의견 제시 → 강도에 맞게 분류
- "논거 종목": 비교/예시로 사용 → 최대 "긍정"까지만. "매수" 분류 금지.
- "뉴스 종목": 시황 전달 → "중립"
- "리포트 종목": 증권사 리포트 전달 → 규칙2 참고
- "교육 종목": 설명/해설 → 시그널 제외 (테이블에서 제외)

예시:
"삼성전자 영업이익 209%↑! 코인 중에는 캔톤만 실적 있다"
→ 삼성전자: 논거 → 🟢긍정 (매수 아님)
→ 캔톤: 결론 → 🔵매수

═══════════════════════
핵심 규칙 2: 리포트 브리핑 구분
═══════════════════════
(a) 리포트 내용만 전달 (본인 의견 없음)
→ mention_type = "리포트", 신호 = "중립"
예: "교보증권에서 목표주가 37만원으로 상향했습니다" (해설만)

(b) 리포트 전달 + 본인 동의/추천 표현
→ mention_type = "결론", 신호 = 강도에 맞게
예: "이 종목 저는 강력 추천드립니다"

(c) 리포트 전달 + 본인 반대 표현
→ mention_type = "결론", 본인 의견 기준으로 분류

(d) 리포트 전달 + 미묘한 긍정 뉘앙스
→ mention_type = "리포트", 신호 = "중립", confidence = "low"
→ reasoning에 "미묘한 동의 뉘앙스 존재하나 명시적 추천 아님" 기록
예: "이수페타시스... 장기 투자자에게 솔깃한 뉴스"

판단 기준: "발언자가 본인 돈을 넣겠다는 뉘앙스가 있는가?"

═══════════════════════
핵심 규칙 3: 전망 vs 매수 권유 구분
═══════════════════════
동사 어미로 판단:

전망 (→ 긍정):
- ~이다/된다/보인다/전망이다/예상된다
- "코스피 7000 갈 수 있다"
- "충분히 가능하다고 봅니다"
- "올라갈 수밖에 없다" (강한 전망도 전망)
- "못 사면 어쩔 수 없다, 사면 좋겠다" (희망 표현)

권유 (→ 매수):
- ~해라/사라/담아라/들어가라/늘려라
- "지금 안 사면 후회한다" (간접적이지만 매수 권유)
- "~하시면 좋겠다/추천드립니다" (공손한 권유)
- "비중 없으면 지금이라도 들어가라"

═══════════════════════
핵심 규칙 4: 지수/섹터/ETF 분류
═══════════════════════
market 필드 (10종): KR / US / JP / CN / CRYPTO / SECTOR / INDEX / ETF / MACRO / OTHER

- INDEX (코스피, S&P500, 나스닥 등):
  → 시그널 테이블에서 제외. 시장 전망으로만 요약에 기록.

- SECTOR (증권주, 건설주, 반도체 섹터 등):
  → 시그널 유지, market=SECTOR 표기
  예: "증권주 사라" → 🔵매수, market=SECTOR

- ETF (TIGER 반도체, KODEX 2차전지, SOSX 등):
  → 시그널 유지, market=ETF 표기
  → 구성종목으로 전파 금지 (ETF 시그널 ≠ 개별종목 시그널)

- MACRO (환율, 금리, 금값 등):
  → 시그널 유지, market=MACRO 표기

- JP/CN/OTHER: 해외 종목 분류 (V7 신규)

═══════════════════════
핵심 규칙 5: 교육/설명형 콘텐츠 구분
═══════════════════════
mention_type = "교육" 판단 기준:
- "~인 이유는", "~의 구조는", "~을 알아보자" = 교육
- 현상 설명, 원리 해설, 역사적 맥락만 제공
- 교육 → 시그널 제외 (테이블에서 제외)

단, 교육 후 "그래서 나는 ~한다" → 결론 (교육 아님)

**V7.1 추가: 교육/투자의견 경계 케이스**
전망 제시 후 즉시 거리두기 표현 → 교육으로 분류:
- "HDC현대산업개발이 좋아지죠. 하지만 교과서적인 이야기고"
- "일반적으로 ~하지만 실제로는 다른 변수가 많습니다"
- 판단 기준: 발언자가 책임회피/거리두기 표현을 동반했는가?

═══════════════════════
핵심 규칙 6: 조건부 발언 처리
═══════════════════════
조건부 발언 감지 시:
- "~한다면", "~될 경우", "이 가격대에서는", "1분기까지는", "만약~"
→ confidence = medium (무조건 하향)
→ reasoning에 조건 원문 포함

═══════════════════════
핵심 규칙 7: 기존 규칙 유지
═══════════════════════
- 영상 1개 × 종목 1개 × 발언자 1명 = 신호 1개
- 반론 소개 후 반박 → 최종 입장 기준
- "~던져라" = 명시적 매도
- "실적이 뭐예요?" (의문 제기) ≠ "팔아라" (매도 권유)

═══════════════════════
핵심 규칙 8: 게스트 채널 규칙
═══════════════════════
- 진행자 질문은 신호 아님
- 게스트 답변만 신호로 분류
- 같은 영상에서 게스트 의견 다르면 각각 별도 신호
- 시황/뉴스 앵커: 기본값 = 뉴스/중립, 명시적 권유 표현 있을 때만 상향
- 패널 토론에서 "저도 동의합니다"는 독립 시그널 아님

═══════════════════════
핵심 규칙 9: 중복 처리
═══════════════════════
같은 발언자가 여러 영상에서 같은 종목 언급:
- 발언자 이름 정규화 후 중복 판단
- 최신 영상 기준 1개만 남김
- 의견 변경 시 최신 채택 + reasoning에 "이전 의견: ~에서 변경됨" 기록

═══════════════════════
핵심 규칙 10: 목표주가 제시 판단
═══════════════════════
- 증권사 리포트의 TP 전달 → 리포트/중립
- 본인이 직접 TP 제시 + 긍정 톤 → 긍정
- 본인 TP + "사라" 표현 → 매수
- TP만 제시 (톤 중립) → 리포트/중립

═══════════════════════
핵심 규칙 11: 제목/썸네일 무시
═══════════════════════
영상 제목과 썸네일은 시그널 판단에 절대 사용하지 않음.
오직 자막(발언 내용)만 기준.

═══════════════════════
핵심 규칙 12: 비교 발언 처리
═══════════════════════
"A보다 B가 낫다" 형태의 상대적 비교:
- 선호 종목(B)만 시그널로 분류
- 비선호 종목(A)은 시그널 제외 (비교 맥락에서만 언급)
- 단, "A는 팔아라"처럼 명시적 매도가 있으면 A도 시그널

═══════════════════════
핵심 규칙 13: 유료 콘텐츠 티저 감지 (V7 신규)
═══════════════════════
다음 패턴 감지 시 mention_type = "티저":
- "자세한 건 유료/멤버십/클럽에서"
- "종목은 구독자 전용"
- "풀버전은 네이버 프리미엄/효친클럽/VIP 등에서"
- 무료 영상에서 섹터/테마만 소개하고 종목 미공개

티저 처리:
- 무료 부분에서 언급된 섹터/테마 → mention_type="티저", signal="중립"
- 종목이 공개되지 않았으므로 시그널 생성 금지
- reasoning에 "유료 전환 티저: 종목 미공개" 기록
- ⚠️ 티저에서 암시된 섹터를 추론하여 종목 시그널을 만들지 말 것

**V7.1 추가: 티저 섹터 힌트 처리**
종목명은 숨기고 섹터만 공개하는 경우:
- "전기차배터리 관련, 반도체장비" + "자세한 건 유료에서"
→ 섹터 정보도 mention_type="티저"로 분류, 시그널 생성 금지
→ 힌트성 섹터 정보는 투자 의견이 아닌 유료 전환용 미끼

예시:
- 이효석: "반도체 다음 4종목 — 자세한 건 효친클럽에서"
  → 시그널 0개. 요약에만 "지주/전력기기/조선/로봇 섹터 언급 (티저)" 기록

═══════════════════════
핵심 규칙 14: 바스켓/섹터 추천 처리 (V7 신규)
═══════════════════════
종목이 아닌 섹터/바스켓 단위 추천:
- "반도체 소부장 끝까지 갖고 가라" → market=SECTOR, signal=해당 강도
- "증권주 ETF 사라" → market=ETF, signal=해당 강도

종목 나열 + 바스켓 추천:
- 발언자가 여러 종목을 나열하며 "이런 바스켓으로 가라"
→ 바스켓으로 1개 SECTOR 시그널 생성
→ 개별 종목은 2분 이상 깊은 분석이 있는 경우에만 별도 시그널

강도 구분 기준:
- 깊은 분석 (2분+ 설명, 논거 제시, TP 언급) → 별도 시그널 (매수/긍정)
- 간단 언급 (이름만, 1줄 설명) → 별도 시그널 없음. 바스켓에 포함

예시:
- 박병창: "한미반도체, 유진테크, PSK — 이런 바스켓으로 가라"
  → 반도체 소부장 SECTOR: 긍정 (1개 시그널)
  → 개별 종목: 시그널 없음 (간단 나열)

- 조진표: "원익IPS — 삼성 설비투자 시 가장 먼저 발주 나오는 종목" (3분 설명)
  → 원익IPS: 매수 (깊은 분석으로 별도 시그널)

═══════════════════════
핵심 규칙 15: 포트폴리오 전략/자산 배분 (V7 신규)
═══════════════════════
"433 전략", "6:4 배분" 등 포트폴리오 전략 제시:
→ 시그널 생성 금지. 요약에만 기록.
→ 전략 내 개별 종목이 별도 추천되었으면 해당 종목만 시그널

예시:
- "40% 대형주, 30% 소부장, 30% 글로벌 — 이렇게 가세요"
  → 시그널 0개 (전략). 요약에 "433 전략 제시" 기록
  → 단, "SK하이닉스는 반드시 담으세요" → SK하이닉스: 매수 (별도 시그널)

═══════════════════════
핵심 규칙 16: 밈/유머/풍자 구분 (V7 신규)
═══════════════════════
다음은 시그널로 분류하지 않음:
- 인터넷 밈/짤 인용: "설명할 시간이 없어 어서타" (밈)
- 풍자/과장 유머: "코스피 만 가자" (유머)
- 과거 사례 농담: "내가 그때 팔았는데 일곱배 갔다" (자조적 유머)
- 시청자 댓글/질문 읽기: "댓글에 삼전 사야 되냐고 하셨는데" (질문 소개)

판단 기준: "진지한 투자 의견인가, 웃자고 한 말인가?"
→ 주변 맥락(웃음, 과장, 비현실적 수치)으로 판단

═══════════════════════
핵심 규칙 17: 과거 보유/매도 경험 (V7 신규)
═══════════════════════
과거시제 보유/매도 발언:
- "나도 갖고 있었는데 팔았다" → mention_type="보유", 시그널 제외
- "예전에 10배 먹었다" → mention_type="보유", 시그널 제외
- "지금도 갖고 있다" + 긍정 톤 → 현재 보유 ≠ 자동 매수. "긍정"까지만.
- "지금도 갖고 있고 더 살 거다" → "매수"

**V7.1 추가: 과거 경험과 현재 의견 혼재 처리**
한 문단에 과거 자랑 + 현재 투자 의견이 섞인 경우:
- "작년에 5만원대에서 사라고 했던 게... 지금이라도 늦지 않다"
- 과거 부분("작년에 사라고 했던") → mention_type="보유", 시그널 제외
- 현재 부분("지금이라도 늦지 않다") → 현재 투자 의견으로 분석
- 판단 기준: 시제 구분하여 현재/미래 표현만 시그널 생성

═══════════════════════
핵심 규칙 18: 카테고리 매도/경계 (V7 신규)
═══════════════════════
특정 종목이 아닌 카테고리 단위 매도/경계:
- "반도체 빼고 나머지 현금화 하라" → market=SECTOR(비반도체), signal=경계
- "코인 전부 팔아라" → market=CRYPTO, signal=매도
- "FOMO 현상이 보인다" → 시그널 아님, 요약에 "시장 과열 경고" 기록

개별 종목 매도와 구분:
- "삼성전자 팔아라" → 삼성전자: 매도 (개별)
- "반도체 빼고 팔아라" → 비반도체 SECTOR: 경계 (카테고리)

═══════════════════════
핵심 규칙 19: 복수 증권사 목표주가 합산 (V7 신규)
═══════════════════════
3개 이상 증권사 TP를 나열할 때:
→ mention_type = "컨센서스", signal = "중립"
→ 평균 TP를 reasoning에 기록
→ 개별 증권사별 시그널 생성 금지 (1개로 합산)

예시:
- "KB 24만, 흥국 24만, 미래에셋 24만, 키움 21만"
  → 삼성전자: 컨센서스/중립 (평균 TP 23.25만원)

단, 발언자가 특정 증권사 의견에 동의 표현:
→ 규칙2(b) 적용, 해당 TP 기준으로 시그널

═══════════════════════
핵심 규칙 20: 시그널 없음 출력 (V7 신규)
═══════════════════════
분석 완료 후 시그널 0개인 경우:
→ "시그널 없음" 명시 출력
→ 영상 요약(STEP 3)은 정상 출력
→ 이유 기록: "교양/교육 콘텐츠", "뉴스 전달만", "유료 티저" 등

예시 출력:
```
시그널: 없음
이유: 교양 채널 — 시황 해설 및 교육 콘텐츠. 특정 종목 투자 의견 없음.
```

════════════════════════════════════
각 신호 출력 형식:
════════════════════════════════════
- speaker: 발언자 이름 (정규화된 이름)
- stock: 종목명 (정규화된 이름)
- ticker: 종목코드 (알면)
- market: KR/US/JP/CN/CRYPTO/SECTOR/INDEX/ETF/MACRO/OTHER (10종)
- mention_type: 결론/논거/뉴스/리포트/교육/티저/보유/컨센서스 (8종)
- signal: 매수/긍정/중립/경계/매도 (5개만!)
- confidence: high/medium/low
- timestamp: 발언 시점 (영상 내 타임스탬프, 예: "12:34")
- key_quote: 핵심 발언 원문
- reasoning: 분류 이유 (논거/결론 근거, 리포트 여부, 전망/권유 구분, 조건부 여부 포함)

════════════════════════════════════
STEP 5: 교차 검증 (14항목)
════════════════════════════════════
1. 논거 vs 결론 구분 맞는지
2. 리포트 전달을 매수로 과대 분류하지 않았는지
3. 시장 전망을 매수 권유로 혼동하지 않았는지
4. 5단계 외 등급 사용하지 않았는지 (STRONG_BUY, HOLD, BUY 등)
5. 같은 발언자+종목 중복 없는지 (정규화된 이름 기준)
6. 진행자 신호가 포함되지 않았는지
7. 발언자 이름 정규화 완료 여부
8. 지수(INDEX) 시그널이 포함되지 않았는지
9. 교육/설명 콘텐츠가 시그널로 잡히지 않았는지
10. 조건부 발언의 confidence가 medium 이하인지
11. 유료 티저에서 종목 시그널을 만들지 않았는지 (V7)
12. 밈/유머를 시그널로 분류하지 않았는지 (V7)
13. 바스켓 나열을 개별 시그널로 분해하지 않았는지 (V7)
14. 시그널 0개일 때 "시그널 없음"을 명시했는지 (V7)

양 많으면 서브에이전트 병렬로 돌려줘.
```
